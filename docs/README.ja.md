## プロジェクト

- Xcode プロジェクト：`Verto.xcodeproj`
- アプリ名：译境(中国語で「翻訳の境地」の意)。中国語以外の UI では「Verto」と表示
- UI 言語:簡体字中国語・英語・日本語・韓国語・スペイン語(iOS 設定のアプリごとの言語で切り替え可能。簡体字中国語がソース言語で、文字列カタログは `Verto/Localizable.xcstrings` + `Verto/InfoPlist.xcstrings`)
- Bundle ID:`com.yspritan.verto`
- 最低 OS:iOS 17
- 技術:SwiftUI、ネイティブ TabView、Observation、AVFoundation、PhotosUI、Speech(SpeechAnalyzer/SFSpeechRecognizer)、Translation。iOS 26+ ではシステムタブバーが自動的に Liquid Glass を採用。
- 権限:音声会話にはマイク権限が必要。iOS 17–25 のフォールバック経路では追加で音声認識権限が必要(両方の用途説明はプロジェクトの INFOPLIST_KEY_* に記載済み)。

## 機能

### テキスト翻訳

原文をタップすると、実物の原文カードが `.spring(duration: 0.45, bounce: 0.12)` のバネ 1 本で静止時の高さからビューポート全体まで展開する。ソフトウェアキーボードは次の runloop でフォーカスを要求し、展開と並行して立ち上がる。タブバーが隠れたりキーボードのセーフエリアが変化したりすると、バネは速度を保ったままリターゲットする——レイアウトの安定を待たず、スナップショットのオーバーレイも終了時のクロスフェード受け渡しも存在しない。テキスト・音声入力・言語変更はまず下書きとして保存され、右上のテラコッタ色の丸いチェック「完成并翻译」(完了して翻訳)をタップして初めてコミットされ、本物の翻訳が発火する。結果画面では言語の入れ替え、読み上げ、コピー、お気に入り、共有、別訳の表示を引き続きサポートする。

**翻訳エンジンとキャッシュ**:テキストタブは Google 翻訳の非公式無料エンドポイント(`translate.googleapis.com`、`client=gtx`、API キー不要)に接続する。送信後はローディング状態を表示し、失敗時は中国語のエラーとリトライボタンを表示。新しい送信は進行中のリクエストをキャンセルする。同一のエンジン・言語ペア・原文による成功結果はプロセス内 LRU(200 件)にキャッシュされ、繰り返しの翻訳は同期的に再利用されネットワークに触れない。失敗はキャッシュされず、リトライは常に本物のリクエストを送る。ソース言語は「自動検出」(`sl=auto`)をサポートし、言語ペアバーに検出結果を表示、検出が確定してから入れ替えが可能になる。単文の訳文には Google が返す別訳が付く(複数文では提供されない。別訳がない場合は「別訳」の入口を非表示)。

### 音声会話翻訳

マイクをタップして聴き取りを開始。話している間、アクティブな吹き出しに volatile な書き起こしと低透明度のリアルタイム粗訳を表示する(350ms スロットルの再翻訳、masked ソーステキスト + generation 番号で期限切れレスポンスを破棄しちらつきを防止)。volatile が 0.9 秒以上安定し、RMS 無音が 0.55 秒以上続くと自動で文を確定する(タップで手動終了も可。55 秒のハードリミット)。

**認識は翻訳を決して待たない**:文の確定は認識ストリーム上の切断点にすぎない(`finalize(through: nil)`)。確定した文は即座に画面に載り(粗訳プレビュー + 翻訳中状態)、正式な翻訳は吹き出しごとに非同期で埋まり、失敗時は吹き出し内でリトライできる。その間も認識は次の文に対して継続し、文境界での取りこぼしはゼロ(トラック状態は消費ベースラインで分割)。自動読み上げは誰も話していない隙間にキューイングされ、再生中は音声入力を一時停止して再取り込みを防ぐ。

**二言語自動検出(デフォルト)**:中央のマイクは言語ペア内の自動識別——言語ごとに 1 本の認識トラックが同じ音声を並行して受け取り、NLLanguageRecognizer の言語確率 + 認識信頼度 + テキスト量のスコアで勝者を選ぶ(1 文字ごとのちらつきを防ぐヒステリシス付き)。検出言語が吹き出しの側と翻訳方向を決めるため、中国語と英語をシームレスに混ぜて話せる。1 トラックの失敗が発話全体を中断することはない(残りのトラックは継続)。言語ボタンをタップすると片側の言語を手動ロック、もう一度タップで自動に戻る。ステータス領域に現在のモードを表示(「正在聆听 · English / 中文」=聴取中、または単一言語)。

**認識スタック**:iOS 26+ かつランタイムで利用可能(`SpeechTranscriber.isAvailable` かつ supportedLocales が非空)なら、SpeechAnalyzer に複数の SpeechTranscriber モジュールを接続して動かす(完全オンデバイス、マイク権限のみ。モジュール失敗時は単一トラックに降格)。それ以外は複数の SFSpeechRecognizer の並行実行にフォールバックする(iOS 17–25 とシミュレータ。両権限が必要)。

**リアルタイム性の要**:認識チェーンはセッション級で永続する——prepare 段階で `.processLifetime` によるモデル常駐と `prepareToAnalyze` のウォームアップで analyzer を構築済みにし、文間は破棄・再構築ではなく `finalize(through: nil)` で切断する(再構築 = 1 文ごとに数秒級のモデルロードを支払う)。TTS 再生や文間の隙間では、一時停止した音声ソースがバッファを破棄することで半二重を維持する(音声セッションを文ごとに setActive でループさせない)。`.fastResults` で最初の volatile を加速。確定しきい値は volatile 安定 0.9 秒 + 無音 0.55 秒。勝者判定は発話開始 0.7 秒以内ならヒステリシスなしで自由に乗り換えられる。

**翻訳ルーティング**:Apple の Translation フレームワークを優先——iOS 26+ では `Translation.TranslationSession(installedSource:target:)` を直接構築(26.4+ では partial 用に別途 `.lowLatency` 戦略のセッションを構築)、iOS 18–25 では AppShell ルートに常駐するホストビュー経由で session を借りる。シミュレータ / iOS 17 / 言語パック未導入 / フレームワークエラー時は自動で Google エンドポイントにフォールバックし、言語ペアごとに判断を記憶する(理由は os.Logger に記録)。

着信・バックグラウンド移行・タブ切り替えはいずれも収音を停止し、会話内容はタブをまたいで保持される(controller は AppShell が保有)。吹き出しには読み上げボタンが付き、ページヘッダー右上には再生モードのクイックメニューがある(設定と同期)。言語ペアの「自動検出」は音声タブでは相手側の言語に基づいて具体的な言語に解決される。final の訳文はプロセス内にキャッシュ(final のみ。partial は LRU に入れない)。final の翻訳失敗は吹き出し内でリトライ可能。波形は実測のマイク音量(vDSP RMS)で駆動される。

### カメラ翻訳

写真選択、認識ローディング状態、メニュー翻訳のオーバーレイカード、フラッシュと露出の状態。

### 言語・履歴・お気に入り

- 言語選択:ソース/ターゲットの切り替え、名前/別名/コードでの検索、選択状態と空結果状態。
- 履歴とお気に入り:共有の翻訳記録、お気に入りフィルタ、即時のスター切り替え、記録タップでテキストタブに再入力。

### 設定と外観

テキストタブ右上の入口から設定画面を開く。翻訳モデルは切り替え可能——Google 翻訳(無料)は現在利用可能、自家製モデルと LLM 翻訳(API キー持ち込み)は「近日公開」の無効プレースホルダとして表示。「音声会話」セクションでは訳文の読み上げ動作を選べる(テキストのみ表示 / 翻訳後に自動読み上げ / ヘッドフォン装着時のみ読み上げ——有線・Bluetooth・USB を含み、ルートをリアルタイム検出)。一般設定には「翻訳後に訳文を自動読み上げ」(テキストタブのみ有効)がある。翻訳エンジン・読み上げモード・設定・最後に使った言語ペアは UserDefaults で永続化され、初回起動ではデモ内容を保持し、以降は記憶した言語ペアで空白から始まる。

**ダークモード**:システムに追従、または設定で外観を手動指定。適応型パレットが全画面・全コンポーネントを貫く。

### ナビゲーションとモーション

- テキスト・音声・カメラはネイティブ TabView の 3 つのトップレベル領域。通常時はタブバーが常に表示され各タブの状態を保持し、テキストタブの集中入力時のみシステムが一時的に隠し、下書きのコミット後に復帰する。iOS 26+ ではシステムが Liquid Glass を描画、iOS 17–25 では対応するシステムタブバー外観を使用。selection が実際に変化したときにシステムの触覚フィードバックを発火する。
- 集中入力状態では `.keyboard` ツールバーに「完了」を置かない。ソフトウェア/ハードウェアキーボードのいずれもページ右上に固定された送信ボタンを使い、下部の操作がシステムタブバーと重ならないようにする。
- 入力トランジションの状態源はただ一つ:下書きが存在するか否か。原文エディタは全過程で同一のアイデンティティを保ち、展開も収納も実物カードへのレイアウトアニメーションである(レンダーツリーが各ビューの frame をフレームごとに補間し、カード面の Shape は毎フレームパスを再計算して 22pt の連続角丸を歪ませない)。ジオメトリ測定・トランザクション跨ぎの検証・フェーズ編成は存在しない。トランジションは全過程で操作可能かつ中断可能で、展開途中にチェックをタップすると現在の速度を保ったまま滑らかに反転する。結果領域は紙面の下で約 0.16 秒の不透明度 transition によりフェードアウト/インし、位置はレイアウトのバネに追従する。チェックは約 40ms の遅延後に 0.84 倍から元のサイズへ軽くポップする。
- 「視差効果を減らす」が有効な場合、レイアウトは直接終状態に切り替わる(サイズ・位置・スケールのアニメーションなし)。結果領域とヘッダーボタンは約 0.12 秒の不透明度フェードのみを保持し、キーボードとタブバーはシステム挙動を継続する。

## 現状とロードマップ

テキスト翻訳は Google 翻訳の非公式無料エンドポイントに接続済み(Google サービスへアクセスできるネットワーク環境が必要)。音声会話は本物の音声認識・翻訳パイプライン(上記参照)。メニュー OCR は現在ローカルのデモデータを使用。自家製モデルと LLM ベースの翻訳エンジンは今後の計画で、設定画面にプレースホルダとして表示中——将来のストリーミング音声翻訳エンジンの継ぎ目は `Verto/Voice/AppleTranslationService.swift` の末尾にすでに用意してある(`StreamingSpeechTranslating` プロトコルスタブ。text→text 層ではなく音声セッション層に接続)。

## Xcode での実行

1. Xcode で `Verto.xcodeproj` を開く。
2. `Verto` スキームを選択。
3. iOS 17 以降の任意の iPhone シミュレータを選択。
4. Run を押す。

ターミナルの `xcode-select` が Command Line Tools や古い Xcode を指している場合は、`DEVELOPER_DIR` を付けてコマンドラインからビルドできる:

```bash
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
xcodebuild \
  -project Verto.xcodeproj \
  -scheme Verto \
  -configuration Debug \
  -sdk iphonesimulator \
  -destination 'generic/platform=iOS Simulator' \
  -derivedDataPath /private/tmp/VertoDerivedData \
  CODE_SIGNING_ALLOWED=NO \
  build
```

## シミュレータの制限

**Apple 公式の制約、実測で検証済み**:SpeechTranscriber と Translation フレームワークはいずれも iOS シミュレータでは動作しない(シミュレータには ANE も翻訳モデルもない)。シミュレータの音声タブは自動的に SFSpeechRecognizer + Google フォールバック経路に落ち、iOS 27 シミュレータでの実測では:**en-US はシステムがローカル認識器を強制するため初期化できず(kLSRErrorDomain 300、オンデバイス/サーバーモードとも失敗)、zh-CN はサーバー認識で完全に動作する**——つまりシミュレータで中国語を話せば「認識→翻訳→読み上げ」を本当に一周でき、英語はマルチトラック自動検出が失敗トラックを静かにスキップする(英語単独トラックの場合は「模拟器暂不支持这种语言的识别」=「シミュレータはこの言語の認識にまだ対応していません」の文言を表示)。

診断は `VertoTests/SpeechAvailabilityProbeTests` でいつでも再実行できる(レポートは /private/tmp/speech-availability-probe.txt に出力)。SpeechAnalyzer 経路、システムのオフライン翻訳、言語モデルのダウンロード、`.lowLatency` 戦略、実機でのデュアルトラック挙動、ヘッドフォン検出は実機でのみ検証可能。UI テストは `--uitest-canned-speech` でスクリプト化された認識と無音 TTS を注入し、本物の音声には一切触れない。

## 自動テスト

プロジェクトには `VertoUITests` UI テストターゲットが含まれ、受け入れ対象はテキスト翻訳とお気に入り、言語検索と選択、音声の「待機 → 聴取 → 確定吹き出し → 一時停止」全フロー、音声読み上げモードの設定選択、カメラ認識結果、ネイティブ TabView のタブ跨ぎ切り替え/選択状態同期/状態保持、「下書き入力 → 完了して翻訳 → 結果画面の復元」、そして DEBUG「視差効果を減らす」終状態リグレッションなどの主要フローをカバーする。

UI テストは一律に `--uitest-canned-translation`、`--uitest-canned-speech`、`--uitest-reset-settings` の起動引数を携帯する:前の 2 つは固定のデモ訳文とスクリプト化された音声認識を注入し(実ネットワーク・マイク・TTS には触れない)、最後の 1 つは永続化された設定をリセットしてアサーションの安定を保証する。

UI は多言語化済みで、テストは一律に簡体字中国語に固定して実行する:共有 Scheme の Test action が `zh-Hans` を指定し(アプリ内でホストされるユニットテストをカバー)、UI テストはさらに起動引数で `-AppleLanguages` を明示的に渡すため、中国語文言のアサーションはシミュレータの言語に左右されない。`LocalizationTests` と英語 UI のスモークテストが各言語リソースの完全性と実際の読み込みを検証する。

ユニットテストは会話コントローラの状態機械(スロットリング、generation 期限切れの破棄、エンドポイントのタイミング、TTS ゲーティング行列、失敗リトライ、キャッシュヒットなど)、翻訳ルーティングのフォールバックチェーン、読み上げモードの永続化、locale マッピングをカバーする。アニメーション可視性のリグレッションは壊れやすいミリ秒級のスクリーンショット比較ではなく、実物の `TextEntryPaperShape.path(in:)` 描画パスに対する DEBUG プローブで、展開と収納が始点・少なくとも 1 つの中間値・終点を通過することを検証する。その他のフローは安定した終状態のみをアサートする。

インストール済みの任意の iPhone シミュレータで実行できる。例:

```bash
xcodebuild test \
  -project Verto.xcodeproj \
  -scheme Verto \
  -configuration Debug \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' \
  -derivedDataPath /private/tmp/VertoTestData \
  CODE_SIGNING_ALLOWED=NO \
  -only-testing:VertoUITests
```

シミュレータは selection が実際に変化したことを検証できるが、物理的な振動は検証できない。触覚の強さと感触は実機の iPhone での最終確認が必要。
